Motivation

My name is Sayz, and I work as a Publisher Development Executive at BuySellAds. I’m known for managing an ad network called Carbon Ads that helps bloggers, designers, and developers to generate ad revenue from their projects.

BuySellAds launched a new ad network last year called Native Performance Network. The network, also based on Monetization API, offers several default ad formats like Flex Bar, FancyBar, and Sticky Box.

As someone who also likes to work on side projects, I understand that creators prefer to have control over how the ad looks like and how it should be integrated to create a better experience for our site visitors.

I wrote Revenue.js because I wanted a quick and simple way to brainstorm different type of ad formats to pitch to the publishers. With Revenue.js, I’ve created hundreds of demos with CodePen and share them with the publishers since early 2018.

HTML & CSS are the most fundamental technology to get started with building any web projects. If you’re new to this field, it can be challenging to customize the ad by calling the API directly. Revenue.js is written so anyone with enough understanding of HTML & CSS can design the ad with ease.

Installation

The easiest way to install Revenue.js on your site is by embedding the following script on before </head> or after <body> opening tag.

<script src="https://m.servedby-buysellads.com/native.js"></script>
<script>
  _native.init("CKYIT2QJ", {
    targetClass: 'revenue-js',
    ignore: 'true'
  });
</script>

Once the script is ready, you can start designing your own ad with HTML and CSS using the revenue-js as the class name. Here is the most basic text ad you can insert in your page:

<div class="revenue-js">
  <style>
  .#native_index# {
    background-color: #native_bg_color#;
    color: #native_color#;
    display: block;
    padding: 10px;
    border-radius: 4px;
  }
  </style>
  <a href="#native_link#" class="#native_index#">
    #native_company# — #native_desc#
  </a>
</div>

Here is the rendered result:

#native_company# — #native_desc#

Options

_native.init accepts two parameters: zone and options.

  • Zone: The key you receive when you’re accepted as a publisher.
  • Options: There are multiple options you can select to customize the values from the framework.
default_options = {
  carbonZoneKey: "",
  ignore: "false",
  placement: "nativejs",
  prefix: "native",
  targetClass: "native-ad"
};
OptionsDetails
carbonZoneKeyYou can use Carbon zone key as the fallback when no Native ads are returned from the API.
ignoreSet the value to true for the development purpose.
placementThe shorthand for your domain. For example, the placement for this site is revenuejscom
prefixThe prefix you’re inserting for the placeholder. The default is native for placeholders like #native_link#. If you change the value to bsa, you must use #bsa_link# as the placeholder.
targetClassThe target class accept the class name of the HTML element you’ll be targeting. Only the content of the HTML inside the target will be replaced with the values returned from the Native API. The default value is native-ad.

Placeholders

The placeholder must be enclosed with hashtag for the framework to detect. For example, if you wish to make the ad looks better by displaying ad background color, you can enter #native_bg_color# inside .native-ad class for the placeholders to be replaced with correct value.

Here are the required value for each ad:

Required PlaceholderDetails
#native_desc#The main description of the ad.
Example: “All the tools your team needs in one place. Slack: Where work happens.”
#native_company#The company name that runs the ads.
Example: “Slack”
#native_link#The URL to track the impressions and earnings

Here are optionals value that should be used along to optimize the ad performance. All the returned color values are hexadecimal.

Text PlaceholderDetails
#native_bg_color#The ad background color
#native_bg_color_hover#The ad background color on hover
#native_color#The #native_desc# font color
#native_color_hover#The #native_desc# color on hover
#native_cta#Call to action, for example “Learn More”
#native_cta_bg_color##native_cta# background color
#native_cta_bg_color_hover##native_cta# background color on hover
#native_cta_color##native_cta# font color
#native_cta_color_hover##native_cta# font color on hover
#native_index#The index of returned ads. It’s useful if you wish to display more than one ad on a single page
#native_title#The ad title

There are other two values that you must request differently. Instead of using img src=#native_logo# to request the image, you have to use img native_src="#native_logo#" to request image to avoid sending empty request to load the image.

Image PlaceholderDetails
#native_small_img#Logo size image URL — 80 × 80 pixels
#native_logo#Medium size image URL — 250 × 100 pixels
